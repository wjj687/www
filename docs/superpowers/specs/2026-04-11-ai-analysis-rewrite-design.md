# 《诗经》AI解析全量重写设计

## 目标

把《诗经》自第 `3` 首起直到第 `305` 首的 `AI解析` 全部重写一遍，使后半部分不再出现“越写越短、越写越薄、越写越像模板”的问题。

这次重写不改页面交互，不改阅读入口，不重做数据骨架，而是在保留现有阅读结构的前提下，把文本本身整体提升到更高的完成度。完成后应达到以下目标：

- 第 `3` 首到第 `305` 首全部完成重写
- 保留现有 `title / subtitle / sections` 数据结构
- 普通篇目至少具备一篇扎实、完整、能成立的长文式赏析
- 重要篇目明显厚于普通篇目，读感上能和前两首保持同一层级
- 语言详实、优美、有文气，但不空泛，不套话，不机械抒情
- 解析重点立足诗本身，兼顾章法、情绪、意象、历来读法与必要背景

## 范围

本次任务包含：

- 重写 [F:\codex\poem\data\ai-analysis.ts](/F:/codex/poem/data/ai-analysis.ts) 中除前两首外的全部 `AI解析`
- 重写所有 `ai-analysis-*.ts` 批次文件中的对应条目
- 统一重写每首诗的：
  - `title`
  - `subtitle`
  - `sections`
- 对前后批次的文风、篇幅、密度进行统一校正

本次任务不包含：

- 改动 `AI解析` 页面布局
- 改动 `AI解析` 的交互方式
- 把解析改成纯长文单字段结构
- 为所有解析补充来源引用显示层
- 改动前两首已经完成度较高的解析正文

## 现状判断

当前 `AI解析` 已经形成完整的数据链路，但存在明显的不均衡：

- 前两首以及少数早期批次文本较厚，具有较明确的阅读节奏与判断力
- 后续批量补写的部分，篇幅逐渐缩短
- 一些条目开始出现批量生成式的平均句长、平均节数和平均判断方式
- 节标题、导语和段落之间的层次关系不够稳定，部分篇目像“分节过后的小短评”，而不像完整赏析

因此，这次重写的重点不是“补齐”，而是“重立标准”。

## 方案比较

### 方案 1：保留现有结构，只把每节写厚

优点：

- 页面和数据结构完全不动
- 风险最小
- 可以直接在现有批次文件上重写

缺点：

- 如果没有额外的写作标准约束，容易继续写成“厚一点的模板”
- 仅靠加字数，不能自动解决后半部分文风走弱的问题

### 方案 2：保留数据骨架，但把正文改成更自由的长文式赏析

优点：

- 不需要改页面层
- 允许重要篇目显著展开
- 允许普通篇目保持完整度但不必强行写成同一种节奏
- 最适合兼顾工程稳定和写作质量

缺点：

- 需要明确的统一写作标尺
- 重写量大，对批次间一致性要求高

### 方案 3：改成纯长文结构

优点：

- 表达最自由
- 最容易形成真正完整的文学评论

缺点：

- 需要改页面读取和展示层
- 会扩大任务范围
- 不符合“先把文本质量提上去”的当前优先级

## 采用方案

采用方案 2：

- 外层继续保留 `title / subtitle / sections`
- 内层按“长文式赏析”重写，不再使用机械均分的小节思路
- 普通篇目和重要篇目采用不同的厚度标准
- 页面不动，数据接口不动，主要提升全部落在文本质量上

## 数据结构约束

### 保留的结构

每首诗继续使用现有结构：

- `poemId`
- `title`
- `subtitle`
- `sections`

其中 `sections` 仍由：

- `title`
- `paragraphs`

组成。

### 重写后的结构策略

不改变字段，但改变每个字段的用途和密度：

- `title`  
  不再只是“篇名 + 一个形容句”，而是要直接提出这首诗最值得进入的阅读角度。

- `subtitle`  
  不再只是摘要式补充，而是承担“把读者带进这首诗的气口”这一职责。可以更有文气，但必须落地，不能空。

- `sections`  
  不再追求统一节数和统一句法。每首诗依然分节，但节与节之间要像同一篇长文中的自然推进，而不是拆开的评论卡片。

## 篇幅标准

### 普通篇目

普通篇目采用以下标准：

- 约 `3` 节
- 每节 `2-3` 段
- 每段必须有明确分析支点，不允许只做气氛抒写

目标效果：

- 至少是一篇完整、扎实、能独立成立的长文式赏析
- 读完后应能知道这首诗写了什么、为什么好、好在何处

### 重要篇目

重要篇目允许并要求明显更厚。包括但不限于：

- 《蒹葭》
- 《氓》
- 《黍离》
- 《采薇》
- 《七月》
- 《生民》
- 《文王》
- 《大明》
- 《玄鸟》
- 《长发》
- 以及各风、雅、颂中公认重量更高或历来歧解更多的篇目

重要篇目建议：

- `3-4` 节
- 每节 `3-4` 段
- 必须有更明确的层次展开，不允许只是在普通篇目上机械加长

目标效果：

- 读感上明显厚于普通篇目
- 至少接近前两首的完成度与密度

## 写作标准

### 总体原则

这次重写的核心，不是“写得漂亮”，而是“写得扎实之后依然漂亮”。

文本应同时满足：

- 详实
- 优美
- 有诗意
- 有判断
- 不空泛
- 不像批量生成文本

### 内容展开顺序

每首诗原则上按以下顺序自然展开，但不强制写成显式提纲：

1. 先把诗本身说清楚  
   交代它到底写了什么，场景怎样起，情绪怎样进入。

2. 再进入真正的分析  
   包括但不限于：
   - 章法递进
   - 意象反复
   - 语气变化
   - 句子为什么成立
   - 某些关键字眼为何有分量

3. 再补入可依凭的解释层  
   以本地已读 PDF 为主干，辅以必要的公开资料核对。重点不是堆知识，而是让判断站得住。

4. 最后再把余韵收出来  
   允许有文气，允许有感发，但必须从诗里长出来，不能悬空拔高。

### 禁止出现的写法

这次重写要刻意避免以下问题：

- 机械三段论
- 每首都写成同一个声口
- 只会反复说“象征了什么”“体现了什么”
- 空泛拔高到文明母题、永恒主题、时代精神
- 没有分析支点的漂亮话
- 为了显得深刻而故作玄虚
- 为了显得全面而平均分配篇幅

### 文风标准

文风上遵循以下要求：

- 有文气，但不用腔调压人
- 允许长句，但长句里必须有清楚推进
- 允许判断，但判断要能落回诗句
- 允许抒情，但抒情不能取代分析
- 允许不同篇目有不同落点，不强行统一口吻

同时参照 [$humanizer-zh](C:/Users/DELL/.codex/skills/humanizer-zh/SKILL.md) 的约束，重点清除：

- 连接词堆叠
- 套路化总结句
- 过度均匀的段落节奏
- 过度拔高和浮泛归因

## 资料使用标准

### 主资料来源

重写时优先使用已经读过的本地材料：

- `poem_search` 下已读 PDF
- `shangxi` 下已读赏析 PDF
- 当前项目里已经整理出的译文、注释、赏析线索

### 辅助资料来源

在本地资料不足、或某些历史读法需要对照时，允许补充公开资料核对，但原则如下：

- 只作为判断支撑，不做页面层引用展示
- 不把公开资料机械改写成“百科解释”
- 不让资料堆积淹没诗本身

### 写法约束

AI解析不是文献综述，因此：

- 不直接复述一大串学者名字
- 不把不同说法简单并排陈列
- 如有歧解，只保留最值得说、最能帮助阅读的那一两种

## 分批执行策略

### 分批单位

继续沿用现有 `ai-analysis-*.ts` 批次文件，不新增第二套平行结构。

每个批次的处理方式统一为：

1. 通读该批次全部篇目当前文本
2. 结合同批诗歌风格和题材统一判断声口
3. 重写这一批的：
   - `title`
   - `subtitle`
   - `sections`
4. 回头抽看该批次首尾篇目，防止前厚后薄

### 批次间一致性控制

为防止越写越薄，必须执行以下约束：

- 每完成一批，都抽读前段、中段、后段已有篇目作对照
- 如果新批次明显比早期样本薄，必须当批返工，不留到最后
- 重要篇目优先加厚，不允许被普通篇目标准拉平
- 后期批次不得退回到“均匀短句 + 均匀节数 + 均匀判断”的批量写法

## 验收标准

本轮重写完成后，应满足以下标准：

- 第 `3` 首至第 `305` 首全部完成重写
- 前后批次不再存在明显的篇幅塌陷
- 普通篇目至少是一篇完整、扎实的长文式赏析
- 重要篇目明显厚于普通篇目
- `title / subtitle / sections` 数据结构保持兼容
- 随机抽读前段、中段、后段，不出现明显模板腔
- `npx tsc --noEmit` 通过
- `npm run build` 通过

## 风险与处理

### 风险 1：303 首体量太大，写到后面再次失衡

处理：

- 批次内自检
- 批次间抽检
- 重要篇目单独加权
- 发现“批量味”即刻返工，不堆积到最后

### 风险 2：写得太长，反而失去阅读支点

处理：

- 每段必须有分析重心
- 不允许只有气氛抒写
- 长不是目标，厚才是目标

### 风险 3：为了有文气而牺牲清楚

处理：

- 判断必须落回诗句
- 抒情必须从分析里长出来
- 遇到复杂篇目，先讲清楚，再讲漂亮

## 实施结果预期

完成后，用户在阅读第 `3` 首之后的任一首 `AI解析` 时，都应感受到以下变化：

- 不再是“补齐感”，而是“写成感”
- 不再是平均、平面、均匀的短评
- 而是一篇真正读过诗、想清楚之后写下来的赏析

前两首不再是孤峰，后面的诗也会逐渐抬到同一条山脊线上。
